# チケットBDD分割フロー

## 概要
`tickets/active/` にある具体化済みのチケットを読み込み、1つ以上の独立した `.feature` ファイル（BDDシナリオ）に分割・展開するフロー。

## 依頼フレーズ
- `このチケットをBDDファイルに分割して`
- `チケット [ID] を実装可能な単位にバラして`

## 実行プロセス（AIへの指示）

### 1. ソースチケットの解析
- 指定された `tickets/active/` 内のファイルを読み込み、`## BDD Scenarios` セクションを抽出せよ。

### 2. ファイル分割と配置
- 各シナリオを、以下のルールで物理ファイルとして作成せよ。
    - **保存先**: `/server/features/[domain_name]/[scenario_slug].feature`
    - **形式**: Gherkin形式 (Feature / Scenario / Given / When / Then)
    - **言語**: 日本語

### 3. 実装優先順位の提示
- 分割したBDDファイルの一覧を表示し、依存関係（例：共通基盤を先に作る等）に基づいた「実装の推奨順序」をユーザーに提示せよ。

### 4. コンテキストの準備
- 各 `.feature` ファイルの冒頭に、関連する ADR 番号をコメントとして付与し、Copilot が実装時に迷わないようにせよ。
