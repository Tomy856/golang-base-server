# テスト実行プロトコル

## 概要
`scripts/run-tests.sh` を使用して、プロジェクト全体のテスト（ユニットテストおよび結合テスト）を一括実行します。

## 依頼フレーズ
- `テストして`
- `テスト実行`
- `テスト実行して`
- `テストを実行して`
- `run-testsして`

## 実行コマンド（例）
```bash
# 必ず /server で実行すること
cd /server
# ログ用ディレクトリを作成（初回のみでOKだが、スクリプトに含めるとより安全）
mkdir -p logs/error
./scripts/run-tests.sh

## 成功判定基準（AIへの追加指示）
- `Total Coverage` が **0.0% より大きい** 数値であることを確認せよ。
- もし `0.0%` の場合は、テストが空振り（実行場所の間違い等）しているため、表示上の Status が PASS であっても、ユーザーには **「テスト失敗（空振り）」** と報告せよ。
