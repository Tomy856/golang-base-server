# BDD完了時の一括更新フロー

## 概要
1つの振る舞い（BDD）の実装が完了し、テストがすべてパスした際に実行する標準フローです。
「生きた仕様書」の鮮度を保ち、ADRへの記録とチケットのクローズを自動化します。

## 依頼フレーズ
- `BDD完了したから締めて`
- `実装終わったのでドキュメント更新して`
- `一括更新フロー実行`

## 実行プロセス（AIへの指示）
1. **静的解析の更新**: `scripts/reload-callvis.sh` を実行し、`project_structure.svg` と `current_symbols.md` を最新化せよ。
2. **Godocの更新**: `scripts/reload-godoc.sh` を実行し、ブラウザ用ドキュメントを再生成せよ。
3. **ADRの自動生成**: `cat-cmd/adr-gen.md` の指示に従い、本チャットの内容を要約して `docs/adr/` に記録せよ。
    - **重要**: 実装中に再利用した既存ドメインや、却下した設計案を漏らさず記載すること。
4. **コンテキストのクリーンアップ**: 
    - 完了したチケット（Markdown/Jira等）を `Done` に移動せよ。
    - 次のチャットで「既存機能」として検出できるよう、最新の `docs/current_symbols.md` を読み直して、今回の変更が反映されているか自己確認せよ。

## 確認事項
- `docs/diagrams/project_structure.svg` が更新されているか。
- `docs/adr/` に新しい意思決定記録（ADR）が作成されているか。
- `http://127.0.0.1:6060/pkg/` で最新のコードコメントが反映されているか。
