# BDD完了時の一括更新フロー

## 概要
BDD実装完了時に実行し、GitHub Copilotが参照する「設計図」を最新化します。

## 依頼フレーズ
- `BDD完了したから締めて`
- `一括更新フロー実行`

## 実行プロセス（AIへの指示）
1. **静的解析の更新**: `scripts/reload-callvis.sh` を実行せよ。
2. **Godocの更新**: `scripts/reload-godoc.sh` を実行せよ。
3. **ADRの記録**: `cat-cmd/adr-gen.md` に従い、既存ファイルへの「追記」か「新規作成」かを判断して記録せよ。
4. **インデックス更新 (New)**: `scripts/update-adr-index.sh` を実行し、`docs/adr/README.md` を最新化せよ。
5. **完了処理**: チケットを `Done` へ移動し、コンテキストをクリアせよ。

## 確認事項（AIへの最終チェック指示）
1. **設計の整合性**: `docs/adr/README.md`（目次）が更新され、今回の実装が「新規ADR」か「既存への追記」として正しく反映されているか。
2. **シンボルの可視化**: `docs/current_symbols.md` に、今回 Copilot が作成した新設関数が漏れなくリストアップされているか。
3. **図解の同期**: `docs/diagrams/project_structure.svg` を開き、新設したパッケージや関数が全体の呼び出しグラフに正しく組み込まれているか。
4. **型定義の公開**: `http://127.0.0.1:6060` で、新設関数のドキュメント（コメント）が「中学生でも分かる表現」で正しく表示されているか。
